# Single-Source Note

A [[concepts/single-source-note]] is a small knowledge artifact created when a source document is worth preserving only for its specific scope, rather than as part of a broader topic cluster. It is useful for capturing isolated posts, observations, or tests that have value but should not be generalized into a larger framework.

This makes the document a strong example of a single-source note because the intended preservation boundary is narrow: it is not asking for related posts, broader context, or a synthesis across multiple documents.

## Why It Matters

Single-source notes help maintain a clean knowledge base by separating isolated artifacts from larger conceptual pages. They support [[concepts/curation-scope]] by making the inclusion rule explicit: save the item, but do not expand its scope unless future documents justify a broader concept page.

## When to Use This Concept

Use a single-source note when:

- A document is relevant but isolated.
- The source contains only one meaningful post or observation.
- The note should be preserved for traceability or future reference.
- There is not enough evidence to create a broader cross-document concept.
